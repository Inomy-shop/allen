# Code of Conduct

Allen is for engineering work: codebases, workflows, agents, traces, and operational decisions. Project discussion should stay useful for people trying to run, debug, or improve Allen.

## Expected Behavior

- Keep feedback concrete: workflow name, page, package, reproduction steps, logs, and tradeoffs.
- Discuss agent behavior, security risks, and product direction with evidence.
- Respect that Allen can touch real repositories and secrets; do not ask people to paste private code or credentials.
- Help keep issues and PRs focused enough for maintainers to act on.
- Give contributors room to ask questions about workflows, workspaces, traces, and integrations.

## Unacceptable Behavior

- Harassment, threats, insults, personal attacks, or discriminatory language.
- Publishing private information, source code, logs, credentials, customer data, or proprietary prompts without permission.
- Encouraging unsafe use of Allen against repositories, secrets, or systems the operator does not control.
- Repeatedly derailing technical threads away from the Allen issue being solved.
- Spam, deceptive reports, or intentionally misleading reproduction steps.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or PRs; block users; or escalate to GitHub moderation.

Security vulnerabilities must be reported through `SECURITY.md`, not public issues.
