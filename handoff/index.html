02 · WHAT'S CHANGING

Key decisions

A summary of the most consequential token shifts. See references/00-design-system.html for the full system.

Surface · navy → off-white

Light-by-default. Optional .dark class ships in the tokens but isn't enabled.

v1 page bg#0A0E1A
v2 page bg#FBFBFA

Accent · cyan → violet

Single accent rule. accent-blue and accent-cyan are aliased to the new violet so legacy JSX still works.

v1 accent#00D4FF
v2 accent#5E6AD2

Type · Inter → Inter Tight + Geist Mono

Inter Tight for prose, Geist Mono for IDs/timestamps/code. Both via Google Fonts or self-hosted.

Chrome · clip-path → 6/10/14px radius

Robotic clipped corners removed. Cards get soft 10px radius, buttons 6px, pills 99px.

Glow shadows → no-ops

shadow-glow-blue/green/red/... all resolve to none. Affordance now lives on borders + hover-color shifts.

Scan-lines removed

.scan-lines::after is a no-op. Existing JSX with that className keeps compiling.

03 · QUICK START

Drop-in install

Phase 0 + Phase 1 from the migration playbook. Total time: ~45 min.
1

Branch the repo

Everything in this handoff is non-destructive — you can revert with one commit. But branch anyway.

$ git checkout -b ui/v2-linear-clean
2

Add fonts

Drop these into packages/ui/index.html <head>:

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Inter+Tight:wght@400;500;600;700&family=Geist+Mono:wght@400;500;600&display=swap" rel="stylesheet">
3

Replace the two token files

From the unzipped handoff folder:

$ cp tokens/index.css.v2          /path/to/allen/packages/ui/src/index.css
$ cp tokens/tailwind.config.js.v2 /path/to/allen/packages/ui/tailwind.config.js
4

Run the dev server

The whole app turns light. Click through every page; expect zero crashes — every legacy class name still resolves.

$ npm --workspace=@allen/ui run dev
5

Hand off to Claude Code

Open Claude Code in the repo and paste:

Read handoff/README.md, MIGRATION.md, COMPONENT_GUIDE.md.
Then for DashboardPage.tsx, open
handoff/references/01-dashboard.html and update the
component to match the D2/Linear-clean artboard.
Use only token classes — no inline hex colors.

05 · DIRECTION RATIONALE

Why Linear-clean

Three directions were explored across all 12 pages. The tokens here implement Direction 02. The other two are kept in references/themes.css for future swap.

D1 · Mission Control

Dark, dense, monitoring-room aesthetic. Heavy data, live counters, telemetry-first.

not implemented

D2 · Linear-clean ✓

Calm, light, generous. Built for people who use Allen alongside their own work — not as a 24/7 console.

implemented

D3 · Operator

Brutalist mono. Everything is a CLI. Maximum density, zero decoration.

not implemented
