Direction 02 · Linear-clean

The Allen design system.

A calm, light, generous product surface. Designed for people who use Allen alongside their own work — not as a 24/7 mission console. Card-first layouts, soft borders, and a single focused accent color carry visual weight so density appears only where it earns its place.

12 · PATTERNS

Design principles

PRINCIPLE 01
Calm by default
Most pages should look almost empty until you fill them. White space is the most underused tool — earn density by exception, not by habit.
PRINCIPLE 02
Mono for the machine
Anything you'd copy-paste — IDs, hashes, file paths, durations, costs, cron — uses Geist Mono. Anything you read prose-style uses Inter Tight.
PRINCIPLE 03
One accent, used sparingly
Violet #5E6AD2 only highlights the active state, the primary action, or a clickable identifier. If everything is violet, nothing is.
PRINCIPLE 04
Status earns saturation
Saturated red, green, blue, and amber are reserved for run states. A red error pill should always mean an actual error — not a tag color.
PRINCIPLE 05
Group, then show
Lists of more than 5 items get a header band with a count. Cards never sit naked on the page — always inside a labelled section or tab.
PRINCIPLE 06
Live, but not loud
Pulses, spinners, and counters should be visible but never demand attention. Use animation to show ‘still working’ — never to celebrate.
