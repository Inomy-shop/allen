# Support

Use the right channel for the Allen problem you are seeing.

## Bugs

Open a GitHub issue with the bug template when an Allen workflow, agent run, workspace, execution trace, API route, integration, or UI page fails.

Include:

- Workflow name, page, or command.
- Repo setup details that are safe to share.
- Execution ID, workspace ID, server logs, browser console output, and screenshots when available.
- Node.js, npm, MongoDB, Claude Code CLI, OS, and browser versions.

Remove secrets, private repo content, customer data, and proprietary prompts before posting.

## Usage Questions

Use GitHub Discussions when enabled for questions about setup, workflow design, agent behavior, workspaces, MCP servers, and integrations.

## Feature Requests

Use the feature request template. Start with the engineering workflow Allen should support, then describe the desired behavior.

## Security

Use `SECURITY.md` for vulnerabilities involving auth, secrets, repo access, command execution, workspace isolation, artifacts, MCP, or integrations.

## Response Priority

Maintainers prioritize:

1. Security issues.
2. Install and startup blockers.
3. Reproducible failures in core workflows.
4. Workspace, terminal, repo-write, or secret-handling bugs.
5. Focused PRs with validation.
6. Feature requests and general questions.
